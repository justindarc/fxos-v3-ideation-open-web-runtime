Mobile App Development Today

  • One app
  • Multiple platforms, separate code base for each
  • Different Device APIs for each platform

Open Web Runtime Of Tomorrow

  • One app
  • Multiple platforms, share SAME codebase for all
  • Standardized Device APIs across ALL platforms

One Codebase To Rule Them All

  • Current products (Firefox Desktop, Firefox Android) can be built on a single, unified Open Web Runtime)
  • Consistent API support across all platforms

Gecko+Device APIs

  • “Portable” Firefox OS
  • Firefox OS as an “app”

Headless Runtime

  • Firefox OS everywhere
  • Devices without screens (wearables)

Extensibility

  • Plug-in architecture

Web Device APIs

  • Firefox OS has many Web Device APIs
  • Most are exclusive to and can’t be used outside of Firefox OS
  • Many are not on the W3C Standards track

Existing Solutions

PhoneGap/Cordova

  • Not a real web runtime
  • Uses the version of WebKit that is bundled with the device OS
  • Injects/overlays its own non-standard Device APIs

node-webkit

  • Specifically for desktop applications
  • Minimal non-standard Device APIs

Open Web Runtime

Significance

What value (the importance, worth or usefulness of something) does your idea deliver?

  • Firefox OS becomes a platform to make applications for all platforms and for the whole web, rather than being just another OS

Technological Advantage

What technological advantage will your idea deliver and why is this important?

  • Mozilla has a head start on the development of Web Device APIs
  • Ability to influence and get existing Web Device APIs on W3C Standards track

Monetary Value

Why would someone invest time or pay money for this idea?

  • Lowers expenses needed to build and maintain separate code bases for each native platform
